¿Qué es este proyecto?

Esta plataforma te permite crear modelos de aprendizaje supervisado sin escribir código. Hay dos tipos de modelos:

Regresión Logística

Para problemas de clasificación: la variable objetivo es categórica (ej: maligno/benigno, tipo de flor). El modelo calcula la probabilidad de pertenencia a cada clase y asigna la más probable.

Regresión Lineal

Para problemas de predicción numérica: la variable objetivo es un valor continuo (ej: precio, temperatura, progresión de enfermedad). El modelo encuentra la recta que mejor se ajusta a los datos.

Parámetros de entrenamiento

Determina qué porcentaje de los datos se reserva para evaluar el modelo. El resto se usa para entrenar. Un valor típico es 20% (0.2). Más datos de prueba dan una evaluación más confiable pero menos datos para aprender.

Controla la complejidad del modelo. C es la inversa de la fuerza de regularización:
  • C pequeño (0.01): modelo más simple, menos overfitting
  • C = 1.0: balance estándar
  • C grande (100): modelo más complejo, más ajuste a los datos

Número de pasos que el algoritmo realiza para encontrar los coeficientes óptimos. Si ves una advertencia de que el modelo no convergió, aumenta este valor (500-1000).

Fija la semilla del generador aleatorio para que los resultados sean reproducibles. Usar la misma semilla siempre produce la misma separación train/test y los mismos coeficientes.
Importante: Antes de entrenar, los datos se escalan automáticamente (media=0, desviación=1). Esto permite comparar la magnitud de los coeficientes entre sí y mejora la convergencia del modelo.

Paso 3: Evalúa el modelo

Después de entrenar, puedes evaluar el rendimiento del modelo con los datos de prueba. Las métricas disponibles dependen del tipo de modelo:

Accuracy (Exactitud)

Proporción de predicciones correctas sobre el total. Útil cuando las clases están balanceadas.

Precisión y Recall

La precisión mide cuántas predicciones positivas fueron correctas. El recall mide cuántos positivos reales fueron identificados.

F1-Score

Media armónica de precisión y recall. Es la mejor métrica cuando hay clases desbalanceadas.

Matriz de Confusión

Tabla que muestra VP, VN, FP, FN. La diagonal son aciertos; fuera de la diagonal son errores.

Curva ROC / AUC

Solo para clasificación binaria. Mide la capacidad de discriminación del modelo. AUC > 0.9 es excelente.

Coeficientes (log-odds)

Indican la dirección e intensidad del efecto de cada característica sobre la probabilidad de la clase.

R² (Coeficiente de Determinación)

Proporción de la varianza explicada por el modelo. R²=1 es predicción perfecta, R²=0 es peor que la media.

RMSE

Raíz del error cuadrático medio. Penaliza errores grandes. Está en las mismas unidades que la variable objetivo.

MSE

Error cuadrático medio. Similar al RMSE pero sin raíz, en unidades al cuadrado.

MAE

Error absoluto medio. Promedio simple de los errores, no penaliza más los errores grandes.

Gráfico de Residuales

Muestra la diferencia entre valor real y predicción. Idealmente los residuales se distribuyen aleatoriamente alrededor de cero.

Reales vs Predichos

Gráfico de dispersión que compara valores reales con predicciones. Entre más cerca de la diagonal, mejor.

Paso 4: Predice nuevos valores

Formulario de predicción

Una vez entrenado el modelo, puedes usarlo para predecir:

  1. Ve a la página de detalle del modelo
  2. Haz clic en Predecir
  3. Completa los campos con los valores de las características
  4. El modelo devolverá:
    • Logística: la clase predicha + probabilidades
    • Lineal: el valor numérico predicho
Notas importantes
  • Los valores se escalan automáticamente al mismo rango usado durante el entrenamiento
  • Usa punto (.) como separador decimal
  • Para clasificación, la clase con mayor probabilidad es la predicción final
  • Para regresión, el resultado es un valor numérico en las mismas unidades que la variable objetivo
Ejemplo: Si entrenaste con el dataset de Diabetes, al predecir con nuevos valores médicos obtendrás la progresión estimada de la enfermedad.

Preguntas frecuentes

Positivo: A mayor valor de la característica, mayor la probabilidad (logística) o el valor predicho (lineal).
Negativo: A mayor valor de la característica, menor la probabilidad o el valor predicho.
La magnitud (valor absoluto) indica la fuerza del efecto. Los coeficientes están en escala estandarizada, por lo que son comparables entre sí.

El overfitting ocurre cuando el modelo se ajusta demasiado a los datos de entrenamiento y no generaliza bien a datos nuevos. Señales de overfitting:
  • Accuracy perfecto o casi perfecto en entrenamiento pero bajo en prueba
  • Coeficientes muy grandes (valores absolutos altos)
  • Cómo evitarlo: aumentar C pequeño (más regularización), aumentar test size, o conseguir más datos

Si ves una advertencia de convergencia, prueba:
  1. Aumentar max_iter (500, 1000 o más)
  2. Verificar que los datos estén escalados (la plataforma lo hace automáticamente)
  3. Reducir el valor de C (más regularización ayuda a la convergencia)

Como regla general:
  • Mínimo: 50-100 muestras para resultados básicos
  • Ideal: Al menos 10 muestras por cada característica
  • Para clasificación: Al menos 10-20 muestras por clase
  • Más datos siempre mejoran la generalización del modelo

El R² (coeficiente de determinación) indica qué proporción de la variabilidad de los datos explica el modelo:
  • R² = 1.0: El modelo explica toda la variabilidad (predicción perfecta)
  • R² = 0.7: El modelo explica el 70% de la variabilidad
  • R² = 0.0: El modelo no explica nada (equivale a predecir siempre la media)
  • R² negativo: El modelo es peor que predecir el promedio
En ciencias sociales/biomédicas, R² > 0.3 se considera aceptable. En física/ingeniería se espera R² > 0.9.
