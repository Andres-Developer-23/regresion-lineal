¿Cómo la regresión lineal se convierte en clasificación?

La regresión lineal produce valores continuos (ej: precio de una casa). Para clasificación necesitamos probabilidades entre 0 y 1.

La regresión logística aplica la función sigmoide a la salida de la regresión lineal:

z = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ (combinación lineal)
P(y=1|x) = σ(z) = 1 / (1 + e-z)

Donde z es la combinación lineal (como en regresión lineal), y σ(z) la transforma a un valor entre 0 y 1.

  • Si P ≥ 0.5 → se clasifica como clase positiva
  • Si P < 0.5 → se clasifica como clase negativa

Para múltiples clases, se entrena un clasificador binario por cada clase (One-vs-Rest).

Interpretación: La curva sigmoide transforma cualquier valor real de z en una probabilidad entre 0 y 1. El punto donde z=0 corresponde a probabilidad 0.5 (umbral de decisión).

¿Cómo aprende el modelo?

El entrenamiento supervisado consiste en:

  1. Alimentar al modelo con datos etiquetados (features + target conocido)
  2. Calcular la predicción con los coeficientes actuales
  3. Medir el error entre la predicción y el valor real usando la función de pérdida (log-loss):
    J(β) = -[y·log(p) + (1-y)·log(1-p)]
    Donde p es la probabilidad predicha e y es la clase real (0 o 1).
  4. Ajustar los coeficientes para minimizar el error, usando descenso por gradiente (cada iteración da un paso hacia el mínimo)
  5. Repetir hasta que el error no disminuya significativamente (convergencia)
Parámetro C (regularización): Controla el balance entre ajustar bien los datos (C grande) y mantener el modelo simple (C pequeño). Un C muy grande puede causar overfitting (memoriza los datos) y uno muy pequeño underfitting (no aprende patrones).
Separación entrenamiento / prueba

Los datos se dividen en dos conjuntos: Entrenamiento (para aprender patrones) y Prueba (para evaluar desempeño en datos no vistos). Esto permite detectar overfitting.

Escalamiento de características: Para que el modelo converja correctamente, las features se escalan usando StandardScaler (media=0, desviación=1).

Métricas principales

Accuracy = (VN + VP) / Total
Proporción de aciertos total. Útil si clases están balanceadas.

Precisión = VP / (VP + FP)
De los que clasificamos como positivos, ¿cuántos lo son realmente?

Recall = VP / (VP + FN)
De los positivos reales, ¿cuántos identificamos?

F1-Score = 2 × (Prec × Recall) / (Prec + Recall)
Media armónica de precisión y recall. Útil con clases desbalanceadas.

Curva ROC y AUC

La curva ROC grafica la Tasa de Verdaderos Positivos vs la Tasa de Falsos Positivos para distintos umbrales.

El AUC (Área bajo la curva) mide la capacidad de discriminación:

  • AUC = 0.5: Aleatorio (sin poder predictivo)
  • 0.7 - 0.8: Aceptable
  • 0.8 - 0.9: Excelente
  • > 0.9: Sobresaliente
La curva ROC solo aplica para clasificación binaria.

Proceso de predicción
  1. El usuario ingresa los valores de las características del nuevo registro
  2. Los valores se escalan usando la misma transformación aplicada a los datos de entrenamiento
  3. El modelo calcula la combinación lineal z = β₀ + β₁x₁ + ...
  4. Se aplica la función sigmoide para obtener probabilidades por clase
  5. Se asigna la clase con mayor probabilidad

Interpretación de probabilidades

Las probabilidades indican qué tan seguro está el modelo de su predicción:

  • P > 0.9: Alta confianza en la predicción
  • 0.6 < P < 0.9: Confianza moderada
  • P < 0.6: Baja confianza (las clases son casi igualmente probables)
Precaución: El modelo solo es confiable para datos similares a los de entrenamiento. Predecir sobre datos muy diferentes (extrapolación) puede dar resultados erróneos.

Principios de uso responsable
  • Esta es una herramienta educativa — Los modelos entrenados aquí no deben usarse para decisiones críticas sin validación adicional.
  • Sesgo en los datos — Si los datos de entrenamiento tienen sesgos (ej: sobrerrepresentación de un grupo), el modelo los heredará y amplificará.
  • Correlación ≠ causalidad — Un coeficiente grande no significa que esa feature cause la clase. Solo indica correlación estadística.
  • Transparencia — Siempre documenta cómo se entrenó el modelo, con qué datos y bajo qué parámetros.
Buenas prácticas
  • Valida siempre el modelo con datos no vistos durante el entrenamiento (test set)
  • No uses el modelo para predecir fuera del rango de los datos de entrenamiento
  • Revisa las métricas — Un accuracy alto puede engañar si las clases están muy desbalanceadas
  • Mantén supervisión humana sobre las decisiones automatizadas
  • Actualiza el modelo periódicamente si los datos cambian en el tiempo
El objetivo principal de esta plataforma es aprender los fundamentos de la clasificación supervisada, no producir modelos listos para producción.
Flujo de Trabajo
