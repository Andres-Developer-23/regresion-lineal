¿Qué significan estos parámetros?
Train / Test Split

Separar los datos permite evaluar si el modelo generaliza. El modelo solo ve los datos de entrenamiento; los de prueba simulan datos "nuevos".

Random State (semilla)

La separación de datos es aleatoria. Fijar la semilla garantiza que obtengas los mismos resultados cada vez que entrenas (importante para reproducibilidad científica).

C (Regularización) — Solo regresión logística

La regularización penaliza coeficientes grandes para evitar overfitting. C es la inversa de la fuerza de regularización:

  • C = 0.01: Regularización fuerte → modelo muy simple, puede ser underfitting
  • C = 1.0: Balance estándar entre ajuste y simplicidad
  • C = 100: Regularización débil → modelo se ajusta más a los datos, riesgo de overfitting

A la izquierda (C pequeño), la frontera es más simple. A la derecha (C grande), la frontera se ajusta más a los datos.
