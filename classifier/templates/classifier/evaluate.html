Explicación detallada de las métricas
{% if model.model_type == 'linear' %}
R² (Coeficiente de Determinación)

Fórmula: R² = 1 — (SS_residual / SS_total)

Indica qué proporción de la varianza de la variable objetivo es explicada por el modelo. R² = 1 significa que el modelo explica toda la variabilidad (predicción perfecta). R² = 0 significa que el modelo no explica nada mejor que la media. Puede ser negativo si el modelo es peor que usar el promedio.

RMSE (Raíz del Error Cuadrático Medio)

Fórmula: RMSE = √(Σ(ŷᵢ — yᵢ)² / n)

Mide el error promedio en las mismas unidades que la variable objetivo. Penaliza más los errores grandes que los pequeños (por elevar al cuadrado). Es la métrica más interpretable para regresión.

MSE (Error Cuadrático Medio)

Fórmula: MSE = Σ(ŷᵢ — yᵢ)² / n

Similar al RMSE pero sin la raíz cuadrada. Está en unidades al cuadrado de la variable objetivo. Se usa como función de pérdida porque es diferenciable y penaliza fuertemente errores grandes.

MAE (Error Absoluto Medio)

Fórmula: MAE = Σ|ŷᵢ — yᵢ| / n

Promedio de los errores absolutos. A diferencia del MSE, no penaliza más los errores grandes. Es más robusta ante outliers (valores atípicos). Siempre es menor o igual que el RMSE.

Gráfico de Residuales

Los residuales (y — ŷ) deben distribuirse aleatoriamente alrededor de cero sin patrones. Si se ve un patrón (embudo, curva), el modelo puede tener heterocedasticidad o no linealidad. El histograma debe ser aproximadamente normal centrado en 0.

Reales vs Predichos

Gráfico de dispersión que compara valores reales con predicciones. Idealmente los puntos se alinean en la diagonal. Desviaciones sistemáticas indican que el modelo subestima o sobreestima ciertos rangos de valores.

Accuracy (Exactitud)

Fórmula: (VP + VN) / (VP + VN + FP + FN)

Proporción de predicciones correctas sobre el total. Es la métrica más intuitiva, pero puede engañar cuando las clases están desbalanceadas (ej: 95% clase A, 5% clase B → un modelo que siempre predice A tendrá 95% de accuracy).

Precisión

Fórmula: VP / (VP + FP)

De todos los elementos que el modelo clasificó como positivos, ¿cuántos realmente lo son? Mide qué tan confiable es cuando predice la clase positiva. Una precisión baja significa muchos falsos positivos.

Recall (Sensibilidad)

Fórmula: VP / (VP + FN)

De todos los elementos positivos reales, ¿cuántos fueron identificados? Mide qué tan completo es el modelo. Un recall bajo significa muchos falsos negativos (se escapan casos positivos).

F1-Score

Fórmula: 2 × (Precisión × Recall) / (Precisión + Recall)

Media armónica de precisión y recall. Es útil cuando hay clases desbalanceadas porque considera tanto los falsos positivos como los falsos negativos. Un F1 alto significa buen balance entre precisión y recall.

AUC-ROC

Área bajo la curva de Tasa de VP vs Tasa de FP

Mide la capacidad del modelo para discriminar entre clases a través de todos los posibles umbrales de decisión. 0.5 = aleatorio, 1.0 = clasificador perfecto. Solo disponible para clasificación binaria.

Clases desbalanceadas

Cuando una clase tiene muchos más ejemplos que otra, el accuracy puede ser engañoso. En esos casos, F1-score y la matriz de confusión son más informativos. También se puede usar el parámetro class_weight para dar más peso a la clase minoritaria (no disponible en esta plataforma).
